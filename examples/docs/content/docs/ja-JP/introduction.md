# はじめに

**Kura は「人間とエージェントのためのナレッジベース」** であり、[June](https://june.build) の上に
構築されています。コンテンツのソースは一つ。人間は洗練された Web サイトを読み、AI エージェントは
構造化された機械可読な方法で問い合わせ・呼び出しを行います。ソースは単一で、決してずれません。

## なぜ Kura なのか

ドキュメントの読み手は変わりつつあります。AI エージェントからのトラフィックがますます増えています。
従来のドキュメントツールは「エージェント向け」を後付け機能として扱いますが、Kura はそれを土台にします。

> 一つのソース → HTML（人間向け）、Markdown と JSON（機械向け）、MCP ツール（エージェントが呼び出す）。
> 重複はゼロです。

<Callout type="tip" title="MDX コンポーネント">
このヒントボックスは MDX の `<Callout>` コンポーネントで書かれています。人間はスタイル付きのカードを
見ますが、エージェント向けの `.md` はクリーンなプレーンテキストです。
</Callout>

## 人間とエージェントの共有ナレッジベース

- **人間**: サイドバー、目次、検索、Markdown コピーのブラウジング体験。
- **エージェント**: すべてのページに `.md` ソース、`/llms.txt` インデックス、呼び出し可能な `/mcp`
  検索ツール。

## 設計原則

1. **依存ゼロの検索エンジン** —— Node / Bun / Deno / Cloudflare Workers で同じコードが動きます。
2. **多言語をデフォルトで** —— デフォルトの埋め込みモデルは bge-m3。
3. **設定より規約** —— Mintlify では後付けが必要なものを、Kura は最初から組み込みます。
